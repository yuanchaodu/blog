# Linux 归档策略

<img src="images/Linux.svg" width="300">

**Linux 归档策略**可以理解为：把不常用但仍需保留的数据，从日常系统中整理、压缩、存放到更合适的位置，便于节省空间、降低风险、方便追溯。

可采用以下策略：

### 1. 按数据类型归档

常见归档对象包括：

* 日志文件：如 `/var/log`
* 业务数据：如历史报表、历史订单、历史生产数据
* 配置文件：如 `/etc` 下的重要配置
* 数据库备份文件
* 用户上传文件、附件、图片等

不同数据类型应采用不同保留周期。

### 2. 按时间周期归档

建议按时间分层管理：

* 近 7 天：保留在线，可快速查看
* 近 1 个月：压缩保存
* 近 3～6 个月：转移到归档目录或专用磁盘
* 1 年以上：转移到备份服务器、对象存储或离线介质

例如：

```bash
/archive/logs/2026/06/
```

### 3. 使用压缩归档工具

常用命令：

```bash
tar -czf app-log-202606.tar.gz /var/log/app/
```

常见格式：

* `.tar`：只打包，不压缩
* `.tar.gz`：压缩速度快，常用
* `.tar.xz`：压缩率高，但速度慢
* `.zip`：跨平台方便

### 4. 自动化归档

可使用 `cron` 定期执行归档脚本。

示例：每天凌晨 2 点执行归档任务。

```bash
0 2 * * * /opt/scripts/archive_logs.sh
```

归档脚本中可包含：

* 查找旧文件
* 压缩文件
* 移动到归档目录
* 删除过期文件
* 记录执行日志

### 5. 设置保留周期

可按合规和业务要求设定：

| 数据类型  |  建议保留周期 |
| ----- | ------: |
| 系统日志  |  3～6 个月 |
| 安全日志  | 6～12 个月 |
| 业务日志  |   1～3 年 |
| 数据库备份 | 3～12 个月 |
| 重要配置  |    长期保留 |

### 6. 校验归档文件

归档后建议生成校验值，防止文件损坏。

```bash
sha256sum app-log-202606.tar.gz > app-log-202606.tar.gz.sha256
```

恢复时校验：

```bash
sha256sum -c app-log-202606.tar.gz.sha256
```

### 7. 控制访问权限

归档文件可能包含敏感信息，应限制权限：

```bash
chmod 600 archive-file.tar.gz
chown root:root archive-file.tar.gz
```

必要时可加密保存。

### 8. 定期测试恢复

归档不是“存起来就完事”。应定期抽查恢复，确认文件可用。

```bash
tar -tzf app-log-202606.tar.gz
```

### 推荐策略模板

可以采用：

**分类归档、定期压缩、异地保存、权限控制、定期校验、到期清理。**

简单说，就是：

> 热数据放在线，温数据压缩保存，冷数据转移归档，过期数据按规则清理。
