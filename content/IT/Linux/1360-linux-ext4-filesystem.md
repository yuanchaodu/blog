# Linux ext4 文件系统

<img src="images/Linux.svg" width="300">

## 什么是 ext4 文件系统

**ext4（Fourth Extended Filesystem）** 是 Linux 中最常用的本地文件系统之一，由 ext3 演进而来，也是很多 Linux 发行版（如 Ubuntu、Debian、CentOS、Rocky Linux 等）的默认文件系统。

可以把文件系统理解为**操作系统管理磁盘数据的一套规则**。磁盘就像一个仓库，而 ext4 就是仓库管理员，它负责：

* 文件如何存放
* 文件如何查找
* 空间如何分配
* 删除后如何回收
* 断电后如何恢复

---

## ext4 的发展历程

Linux 文件系统的发展大致如下：

```text
Minix FS
    │
    ▼
 ext (1992)
    │
    ▼
 ext2 (1993)
    │
    ▼
 ext3 (2001)
    │
    ▼
 ext4 (2008)
```

各版本特点：

| 文件系统 | 特点                   |
| ---- | -------------------- |
| ext2 | 简单、高效，没有日志功能         |
| ext3 | 增加 Journal（日志），提高可靠性 |
| ext4 | 更大容量、更快速度、更高稳定性      |

目前绝大多数 Linux 服务器仍大量使用 ext4。

# ext4 的主要特点

## 1. Journaling（日志机制）

这是 ext4 最重要的特点。

例如：

用户正在保存一个 10GB 文件。

如果突然断电：

没有日志：

```text
文件写了一半
目录也改了一半
系统不知道哪里写成功
文件系统可能损坏
```

有 Journal：

```text
先记录：

我要写哪些数据

↓

真正写数据

↓

成功以后删除日志
```

如果断电：

```text
启动时读取 Journal

继续完成

或者回滚
```

因此：

> ext4 很少需要像 Windows 早期那样长时间扫描磁盘。

---

## 2. Extents（连续空间管理）

这是 ext4 相比 ext3 最大的改进之一。

### ext3

一个大文件：

```text
Block1
Block2
Block3
Block4
...
```

每一个 Block 都要记录。

例如：

```
100万个 Block
100万条记录
```

管理成本非常高。

---

### ext4

采用 Extent。

例如：

```
100000 ~ 200000
```

表示：

```
连续 100001 个 Block
```

只需要：

```
起始Block
长度
```

即可。

优点：

* 元数据更少
* 查找更快
* 连续读写更快
* 碎片更少

---

## 3. Delayed Allocation（延迟分配）

传统方式：

```
应用写数据

↓

立即分配磁盘块

↓

写磁盘
```

ext4：

```
应用写数据

↓

先放缓存

↓

等一会

↓

统一规划空间

↓

再写磁盘
```

好处：

* 连续空间更多
* 减少碎片
* 提高写入速度

---

## 4. Multi-block Allocation（多块分配）

以前：

```
一次申请：

1 Block
```

现在：

```
一次申请：

100 Block
```

减少系统调用。

提高性能。

---

## 5. 支持更大的文件

典型限制：

| 项目   | ext3 | ext4                       |
| ---- | ---- | -------------------------- |
| 单文件  | 2TB  | 16TB（典型配置，可更大）             |
| 文件系统 | 16TB | 最多约 1EB（理论值，实际受块大小和内核实现限制） |

对于今天的大容量硬盘来说，ext4 基本足够。

---

## 6. 更快的 fsck

fsck：

```
File System Check
```

即：

```
磁盘检查
```

ext4 保存了更多元数据。

因此：

```
几十TB磁盘

检查速度
远快于 ext3
```

# ext4 的磁盘结构

可以简单理解成：

```text
+---------------------+
| Super Block         |
+---------------------+
| Group Descriptor    |
+---------------------+
| Block Bitmap        |
+---------------------+
| Inode Bitmap        |
+---------------------+
| Inode Table         |
+---------------------+
| Data Blocks         |
+---------------------+
```

分别介绍。

---

## Super Block（超级块）

超级块保存整个文件系统的信息。

例如：

* Block 大小
* inode 数量
* Block 数量
* UUID
* 文件系统状态
* 挂载次数
* 是否需要 fsck

查看：

```bash
sudo dumpe2fs /dev/sda1 | less
```

或者：

```bash
sudo tune2fs -l /dev/sda1
```

---

## Block（数据块）

磁盘不会按字节管理。

而是：

```
Block
```

例如：

```
4KB
```

一个文件：

```
13KB
```

实际占：

```
4KB
4KB
4KB
4KB
```

共：

```
16KB
```

最后一块没有写满。

---

## Inode

Linux 最重要的数据结构。

**Inode 不保存文件名。**

它保存：

* 权限
* UID
* GID
* 时间
* 文件大小
* Block 地址
* Extent 信息
* 链接数

例如：

```
ls -li
```

输出：

```
123456 test.txt
```

其中：

```
123456
```

就是 inode 编号。

---

## Directory（目录）

目录实际上也是文件。

里面保存：

```
文件名

↓

inode
```

例如：

```
hello.txt

↓

inode 1357
```

因此：

```
文件名
≠
inode
```

这也是为什么：

Linux 可以：

```
多个文件名

↓

同一个 inode
```

即：

**硬链接（Hard Link）**。

# ext4 如何存储文件

例如：

```
report.pdf
```

流程：

```
目录

↓

找到：

report.pdf

↓

inode=5321

↓

inode

↓

Extent

↓

Block

↓

真正数据
```

示意图：

```text
文件名
    │
    ▼
Directory Entry
    │
    ▼
inode
    │
    ▼
Extent
    │
    ▼
Data Block
```

# ext4 常用命令

| 命令          | 作用           |
| ----------- | ------------ |
| `mkfs.ext4` | 创建 ext4 文件系统 |
| `fsck.ext4` | 检查并修复文件系统    |
| `tune2fs`   | 修改 ext4 参数   |
| `dumpe2fs`  | 查看超级块信息      |
| `resize2fs` | 调整 ext4 大小   |
| `e2label`   | 修改卷标         |
| `debugfs`   | 调试 ext4      |

例如：

创建：

```bash
mkfs.ext4 /dev/sdb1
```

查看 UUID：

```bash
blkid
```

查看挂载：

```bash
df -Th
```

查看 inode 使用：

```bash
df -i
```

# ext4 的优缺点

### 优点

* 成熟稳定，经过长期生产环境验证。
* 性能优秀，适合大多数服务器和桌面场景。
* 日志机制降低断电后的文件系统损坏风险。
* 工具链完善，维护和恢复较方便。
* 兼容性好，几乎所有 Linux 发行版都支持。

### 缺点

* 不支持像 OpenZFS 或 Btrfs 那样的写时复制（Copy-on-Write）和原生快照。
* 不具备端到端数据校验能力，难以及时发现静默数据损坏。
* 不支持透明压缩等高级特性。
* 不包含集成的卷管理功能，通常需要配合 LVM 使用。

## 通俗理解

可以把 ext4 想象成一本管理有序的档案柜：

* **目录**：档案索引，记录文件名对应的位置。
* **inode**：每份档案的详细信息卡，记录权限、大小、时间等元数据。
* **Extent**：连续存放档案的货架区域，减少碎片并提升读写效率。
* **Journal（日志）**：管理员的操作记录本，发生断电等异常时，可帮助恢复到一致状态。

正是依靠这些机制，ext4 在性能、稳定性和易维护性之间取得了良好的平衡，因此至今仍是 Linux 环境中应用最广泛的文件系统之一。
